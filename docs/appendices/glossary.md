# Glossary

This glossary defines key terms and concepts used in the Shadowing Learning project.

## 🎯 Core Concepts

### Shadowing Learning
A language learning technique where learners repeat speech immediately after hearing it, mimicking the native speaker's pronunciation, rhythm, and intonation.

### Audio Transcription
The process of converting spoken language into written text using speech recognition technology.

### Text Normalization
The process of standardizing text format, including punctuation, capitalization, and grammatical corrections.

## 🏗️ Technical Terms

### Next.js
A React framework for building server-rendered and static web applications.

### React
A JavaScript library for building user interfaces, particularly single-page applications.

### TypeScript
A strongly typed programming language that builds on JavaScript, adding static type definitions.

### shadcn/ui
A collection of modern, accessible React components built with Radix UI and Tailwind CSS.

## 🗄️ Data Storage

### IndexedDB
A low-level API for client-side storage of significant amounts of structured data.

### Dexie
A minimalist wrapper for IndexedDB that provides a more developer-friendly API.

### Blob (Binary Large Object)
A data type used to store binary data, such as audio files, images, or other media.

## 🎵 Audio Processing

### Audio Chunk
A segment of an audio file processed separately, typically 45 seconds with overlap for accuracy.

### Overlap
The portion of an audio chunk that overlaps with the previous/next chunk to ensure continuity.

### Word Timestamp
Precise timing information indicating when each word starts and ends in the audio.

### A-B Loop
A playback feature that repeats a specific segment of audio between points A and B.

## 🤖 AI/ML Services

### Groq
A cloud platform providing fast AI inference, including the Whisper model for speech recognition.

### Whisper-large-v3
OpenAI's advanced speech recognition model for accurate audio transcription.

### OpenRouter
A service providing access to various Large Language Models (LLMs) for text processing.

### LLM (Large Language Model)
AI models trained on vast amounts of text data for natural language understanding and generation.

## 📱 User Interface

### Component
Reusable UI elements that encapsulate functionality and appearance.

### Hook
Special React functions that let you use state and other React features in functional components.

### State Management
The process of managing and maintaining application state across components.

### Responsive Design
Design approach that ensures applications work well on various screen sizes and devices.

## 🔧 Development Concepts

### API (Application Programming Interface)
A set of protocols and tools for building software applications.

### Endpoint
A specific URL where an API can be accessed.

### Environment Variable
Variables that store configuration settings and sensitive information outside of code.

### Middleware
Software that connects different applications or components, often for preprocessing requests.

## 🧪 Testing

### Unit Test
Tests that verify individual functions or components in isolation.

### Integration Test
Tests that verify how multiple components work together.

### End-to-End Test
Tests that simulate real user scenarios from start to finish.

### Mocking
Creating simulated versions of dependencies for testing purposes.

## 🚀 Deployment

### Build Process
The process of converting source code into a production-ready application.

### Static Site Generation
Pre-rendering pages at build time for better performance.

### Server-Side Rendering
Rendering pages on the server for each request.

### Client-Side Rendering
Rendering pages in the browser after JavaScript loads.

## 📊 Data Models

### FileRow
Database schema for storing audio file metadata and binary data.

### TranscriptRow
Database schema for storing transcription results and processing status.

### Segment
Individual sentence units with timing information and annotations.

### WordTimestamp
Precise timing data for each word in a transcription.

## 🔒 Security

### XSS (Cross-Site Scripting)
Security vulnerability that allows attackers to inject malicious scripts.

### CORS (Cross-Origin Resource Sharing)
Security feature that controls how web pages in one domain can interact with resources in another.

### API Key
Secret token used to authenticate requests to API services.

### Input Sanitization
Process of cleaning and validating user input to prevent security vulnerabilities.

## 🎨 Styling

### Tailwind CSS
Utility-first CSS framework for rapidly building custom user interfaces.

### CSS-in-JS
Pattern of writing CSS within JavaScript code.

### Design System
Collection of reusable components guided by clear standards.

### Theme
Coordinated set of colors, fonts, and design elements.

## 📈 Performance

### Optimization
Process of improving application speed and efficiency.

### Lazy Loading
Loading resources only when they are needed.

### Code Splitting
Breaking code into smaller chunks for better loading performance.

### Caching
Storing frequently accessed data for faster retrieval.
